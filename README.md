# 依恋类型自测单页

这个文件夹可以直接作为 GitHub Pages 静态网站使用。GitHub 官方说明 GitHub Pages 可以从仓库发布 HTML、CSS 和 JavaScript 静态文件：https://docs.github.com/en/pages/getting-started-with-github-pages/what-is-github-pages

## 放到 GitHub

1. 新建一个 GitHub 仓库。
2. 把 `index.html`、`README.md`、`THIRD_PARTY_NOTICES.md` 上传到仓库根目录。
3. 进入仓库 `Settings` -> `Pages`。
4. `Build and deployment` 选择 `Deploy from a branch`。
5. Branch 选择 `main`，目录选择 `/root`，保存。
6. 稍等几分钟，GitHub 会给出一个 `https://用户名.github.io/仓库名/` 的网址。

## GitHub 能不能直接算分

可以。GitHub Pages 只负责托管静态文件，真正的计分由 `index.html` 里的 JavaScript 在访问者浏览器中完成。这个版本不需要后端，也不会上传答题数据。

## 计分说明

结果包含四项：

- A 安全型依恋得分
- B 焦虑型依恋得分
- C 回避型依恋得分
- D 未解决型依恋得分

计分方式：每题看所选答案编号是否出现在 A/B/C/D 栏中；出现在哪一栏，该栏就加 1 分。同一题可能同时命中多个栏。第三个选项“两者皆非”按 0 分处理。

计分表参考豆瓣帖《分享一个依恋类型测试（附统计方法）》：https://www.douban.com/group/topic/490193528/ 。题目文字由用户提供。本页面仅供自我理解参考，不是心理诊断。来源说明见 `THIRD_PARTY_NOTICES.md`。
